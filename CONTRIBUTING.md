# Contributing to dotnet-plugin-engine

Thank you for your interest in contributing!

## How to Contribute
1. **Fork the repo** and clone it.
2. **Create a branch** for your feature or fix (`feature/name` or `fix/name`).
3. **Write code** following existing conventions, ensuring high quality.
4. **Add tests** for your changes.
5. **Run tests**: Ensure all pass with `dotnet test`.
6. **Format code**: Run `dotnet format`.
7. **Submit a Pull Request** describing your changes clearly.

## Guidelines
- Follow C# conventions and use `async/await`.
- Include XML documentation for all new public members.
- Keep PRs focused on a single issue.
- Maintain existing author headers.

## Issues
Search existing issues before reporting a new one. Provide a clear description and reproduction steps when reporting bugs.
