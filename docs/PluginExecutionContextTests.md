# PluginExecutionContextTests

Unit tests for `PluginExecutionContext` verifying initialization, state transitions, data handling, and summary generation.

## API

### `NewContext_HasUniqueExecutionId`
Verifies that a newly created context generates a unique execution ID that differs from previously created contexts.

### `NewContext_StateIsRunning`
Ensures that a new context is initialized with the `Running` state.

### `NewContext_StartedAtUtcIsRecentlySet`
Confirms that the `StartedAtUtc` timestamp of a new context is set to the current UTC time.

### `NewContext_CompletedAtUtcIsNull`
Validates that the `CompletedAtUtc` timestamp of a new context is initially `null`.

### `NewContext_DataDictionaryIsEmpty`
Checks that the data dictionary of a new context is empty upon initialization.

### `CompleteSuccess_SetsStateToCompleted`
Tests that calling `CompleteSuccess` transitions the context state to `Completed`.

### `CompleteSuccess_SetsCompletedAtUtc`
Ensures that `CompleteSuccess` sets the `CompletedAtUtc` timestamp to the current UTC time.

### `CompleteSuccess_WithResult_StoresResult`
Verifies that `CompleteSuccess` stores the provided result in the context.

### `CompleteSuccess_DurationIsPositive`
Confirms that the duration calculated after `CompleteSuccess` is a positive value.

### `CompleteFailed_SetsStateToFailed`
Tests that calling `CompleteFailed` transitions the context state to `Failed`.

### `CompleteFailed_StoresException`
Ensures that `CompleteFailed` stores the provided exception in the context.

### `CompleteFailed_SetsCompletedAtUtc`
Verifies that `CompleteFailed` sets the `CompletedAtUtc` timestamp to the current UTC time.

### `Cancel_SetsStateToCancelled`
Tests that calling `Cancel` transitions the context state to `Cancelled`.

### `Cancel_SetsCompletedAtUtc`
Ensures that `Cancel` sets the `CompletedAtUtc` timestamp to the current UTC time.

### `GetSummary_AfterCompleteSuccess_IsSuccessfulTrue`
Validates that `GetSummary` returns a successful result after `CompleteSuccess` is called.

### `GetSummary_AfterCompleteFailed_IsSuccessfulFalse`
Ensures that `GetSummary` returns an unsuccessful result after `CompleteFailed` is called.

### `GetSummary_ContainsPluginName`
Confirms that the summary generated by `GetSummary` includes the plugin name.

### `GetSummary_ToStringContainsKeyInfo`
Verifies that the string representation of the summary contains key information.

### `Data_CanStoreArbitraryValues`
Tests that the data dictionary can store arbitrary key-value pairs of supported types.

### `Metrics_ArePresentOnNewContext`
Ensures that a new context includes an initialized metrics dictionary.
